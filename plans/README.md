## Dependency notes

## Findings considered and rejected

- Cache only the Eloquent relationships: rejected because implicit route model
  binding would still query `books.path` on every request and keep Serverless
  Postgres awake.
- Cache the complete HTTP/Inertia response: rejected because responses contain
  request-specific shared props, cookies, flash data, and authentication state.
  Only the anonymous book data bundle should be cached.
- Block all crawlers immediately: deferred until request/user-agent evidence
  identifies abusive traffic. Legitimate indexing is useful for public book
  pages, and data caching addresses the database-cost problem without removing
  discoverability.
- Add a new caching package: rejected because Laravel's existing Redis cache is
  sufficient and project dependencies must not change without approval.
